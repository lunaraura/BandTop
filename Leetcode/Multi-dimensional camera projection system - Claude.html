Key design questions

1D/2D/3D/4D unification: Your squareMap is already a 16-vertex 4D hypercube (tesseract), which is great because it means a single data structure can serve all configurations — you just project down. The camera's job in each mode:

  • 1D: cast rays along a single axis, sample the object as a function of one parameter — output is basically a line of intensity/depth values
  • 2D: classic 2D raycasting (like a top-down Wolfenstein-style caster) — rays fan out in a plane, hit edges of a square, return distances
  • 3D: rays fan out in 2D (screen x,y) into 3D space, intersecting faces of a cube
  • 4D: this is the interesting one — do you want true 4D raycasting (rays as 4D vectors hitting hypercube "cells"), or a projection of the 4D object down to 3D/2D first, then normal raycasting?

2. "Point cloud" terminology: You have maxCloudPoints — implying the camera samples the object's surface as a cloud rather than doing analytic ray-face intersection. That's a reasonable simplification (much easier than computing real N-D ray/hyperplane intersections) but I want to confirm that's the intent rather than full analytic raycasting.
